## 🌍 Version 2.7.0 — Sections globales

Partagez une section entre plusieurs pages : modifiez-la une seule fois, le changement s'applique partout !

### ✨ Nouveautes

- **Sections globales** — Creez des sections partagees entre vos pages. Contrairement aux modeles qui copient les donnees, une section globale est une reference unique.
- **CRUD dedie** — Un nouvel espace **Contenu → Sections globales** permet de creer, modifier et supprimer les sections globales.
- **Integration dans le constructeur** — Les sections globales apparaissent dans le modal d'ajout de section, avec un style dedie (bordure verte, badge « Globale »).
- **Affichage verrouille** — Les sections globales integrees dans une page sont non editables directement. Un lien permet d'ouvrir la section globale dans un nouvel onglet.
- **Conversion rapide** — Convertissez une section existante en section globale en un clic depuis le menu d'actions.
- **Indicateur d'utilisation** — Lors de la modification d'une section globale, un indicateur affiche les pages qui l'utilisent.
- **Toggle de fonctionnalite** — Activez ou desactivez la fonctionnalite via la cle `pages_global_sections`.

### 🔧 Details techniques

- Nouvelle table `global_sections` (name, section_type, data JSON).
- Les pages stockent une reference `{ "type": "__global", "data": { "global_section_id": ID } }`.
- Helper `resolve_sections($page)` pour le rendu frontend (charge les sections globales en batch).
- Permissions reutilisees depuis le module Pages (view/create/edit/delete pages).
